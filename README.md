# MATLAB Code for Deterministic Learning-based Knowledge Fusion

This repository provides MATLAB code and benchmark data for reproducing the main experiments in:

> New Results of Deterministic Learning-based Knowledge Fusion  


The code was reorganized from the original `papercode/` folder into a reviewer-friendly reproduction package. The original long scripts are kept in `scripts/legacy/`, while the main entry points are placed in `scripts/`.

## Quick Start

1. Open MATLAB.
2. Set this repository root as the current folder.
3. Run:

```matlab
addpath(genpath(fullfile(pwd, "src")));
run("scripts/run_table1_cascaded_tanks.m");
```

To reproduce the Rossler main figures:

```matlab
addpath(genpath(fullfile(pwd, "src")));
run("scripts/run_rossler_main_figures.m");
```

To run the main table and Rossler figure scripts together:

```matlab
run("scripts/run_all.m");
```

## Reproducing Main Manuscript Results

| Manuscript item | Main script | Output | Notes |
|---|---|---|---|
| Cascaded Tanks Table 1 | `scripts/run_table1_cascaded_tanks.m` | `results/generated/table1_cascaded_tanks_results.csv` | Runs DLM-Mean, DLM-Off, DLM-LS, DLM-WLSR1, and DLM-WLSR2 |
| Cascaded Tanks source figures | `scripts/export_cascaded_tanks_source_figures.m` | `figures/exported/cascaded_tanks_prediction_comparison.pdf` | Rebuilds a combined plot from the included `.fig` files |
| Rossler main figures | `scripts/run_rossler_main_figures.m` | `figures/exported/rossler_main_*.png/.pdf/.fig` | Uses `scripts/legacy/NNpaperfigure.m` |
| Rossler sensitivity analysis | `scripts/run_sensitivity_analysis.m` | `figures/exported/rossler_sensitivity_*.png/.pdf/.fig` | Long-running hyperparameter sweep |


The Cascaded Tanks entry point uses `src/experiments/runCascadedTanksExperiment.m`, which refactors the five original system-identification scripts into one method-based runner.

## Environment

Tested environment:

- MATLAB R2020a
- Windows 10/11

Required MATLAB functionality:

- `ode45`
- `pinv`, `qr`, `svd`
- `writetable`
- `savefig`
- `smooth` from Curve Fitting Toolbox is used when available. The wrapper `src/utils/smooth_signal.m` falls back to `movmean` if `smooth` is unavailable.

GNU Octave compatibility is not guaranteed.

The expected RMSE values above were generated with MATLAB R2020a using MATLAB's `smooth` function. Each Cascaded Tanks method preserves the smoothing windows from its original legacy script. If the fallback `movmean` path is used, small numerical differences may occur.

## Data

The Cascaded Tanks benchmark data are stored at:

```text
data/cascaded_tanks/dataBenchmark.mat
```

Variables:

| Variable | Size | Meaning |
|---|---:|---|
| `uEst` | 1024 x 1 | Estimation input |
| `yEst` | 1024 x 1 | Estimation output |
| `uVal` | 1024 x 1 | Validation input |
| `yVal` | 1024 x 1 | Validation output |
| `Ts` | 1 x 1 | Sampling time |

Rossler trajectories are generated synthetically by the MATLAB scripts; no external Rossler dataset is required.

## Metrics

For the Cascaded Tanks benchmark, the table script reports:

- estimation-set free-run RMSE;
- validation-set free-run RMSE;
- RMSE of the one-step output difference, included as a scale reference.

For the Rossler experiments, the legacy main script reports trajectory approximation error and surface-level generalization error. The original code uses a valid-region filter of `Z < 0.3` before averaging the surface error. This filtering rule is preserved for paper traceability and documented in `docs/experiment_protocol.md`.
